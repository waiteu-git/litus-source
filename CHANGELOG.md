# Changelog — リタス（Litus）

東京理科大生向け非公式アプリ「リタス」の変更履歴。
事前登録はこちら → https://lms.waiteu.dev/app

## v1.0.0（2026-07-07 ビルド・初版公開に向けた実験配布）

初回リリース。LETUS課題の自動収集・締切通知、CLASS時間割、モバイル出席登録のワンタップ化をスマホ単体で提供する。

### 追加

- **初回チュートリアル**: 初回起動でアプリの使い方（課題見張り・出席ワンタップ・時間割と通知）を3枚のスライドで紹介し、そのままTUSアカウントでログイン（認証情報は保存しません）
- **起動画面**: 2回目以降の起動はリタスの起動画面でCLASS/LETUSのログイン状態を自動確認してから入場。以後の出席・課題・通知がスムーズに
- **課題の自動更新**: アプリ起動後、バックグラウンドでLETUSの課題を自動収集し、一覧と通知を最新化
- **出席ページへの自動到達**: 出席タブを開くとCLASSの入口→ポータル→モバイル出席登録まで自動で遷移。認証コードを入れて押すだけ
- **通知チャンネル**: Androidの通知チャンネル「出席リマインド」（最優先・ヘッドアップ表示）と「課題リマインド」を分離。アプリを開いている間も通知が表示されるように

### 追加（v2ビルド・2026-07-07 実機フィードバック反映）

- **セットアップ全自動化**: ログインすれば時間割（起動フェーズで自動取り込み）とコース・課題（入場後に自動同期）が操作ゼロで揃うように
- **ログイン導線の修正**: 初回チュートリアル後、CLASSの選択画面を経由せず直接Microsoftログインへ（「ログインへ進む」で確実にログイン画面が出る）
- **「別の画面で操作された」エラーの根絶**: CLASSの画面はアプリ内で常に1枚だけ開く方式に変更（タブ間の競合が原理的に発生しない）＋万一の検知時は自動やり直し
- **出席タブの高速表示**: タブに戻ったとき前回の受付状況を表示したまま裏で更新

### 追加（v5ビルド・2026-07-08）

- **時間割・コースの自動更新**: 起動時にCLASS画面を見せず裏で時間割を更新（1日1回）。コース・課題も引き続き裏で自動同期
- **シラバスを読みやすく**: シラバスをアプリ独自の画面で表示（スマホで見づらい原本HTMLを整形）
- **課題読み込みの高速化**: 2回目以降は締切が前後2週間の課題だけを再確認して同期を短縮
- **課題の手動追加**: アプリ内でLETUSの課題ページを開くと「＋この課題を追加」で追跡対象に追加可能
- **提出状態の取得改善**: 収集時のブラウザ判定を調整し「状態不明」になりにくく
- タブ選択時のハイライトを角丸に

### 安定性

- CLASSの「システムエラー」（セッション競合）を検知して自動復帰
- 長時間放置→復帰時にログイン状態とページを自動で立て直し、通知予約も貼り直し
- 収集がページ読込で詰まった場合の15秒タイムアウト保険

### 追加（v21ビルド・2026-07-09 ホームタブ導入）

- **ホームタブ新設**: タブを「時間割 / ホーム / 課題」の3本に集約。出席・インフォ・設定はホームの中へまとめ、ホームは今後拡充していく起点に
- **出席をホームの中へ**: 出席機能はホームから開く形に。CLASS画面（WebView）はアプリ内で1枚だけをアプリ全体で保持し続けるため、画面を行き来しても再ログインや「別の画面」エラーが起きにくく、開いた瞬間から表示
- **出席通知からの即オープン**: 出席リマインド通知をタップしてアプリを開くと、そのまま出席画面へ直行（起動直後のタップ・起動中のタップ両対応）
- **出席お知らせバナー**: 通常起動時、時間割上で授業がある／CLASSが出席を受付中のときだけ、ホーム上部に「◯◯ 出席登録受付中」などのバナーを表示。数秒後は右下の小さいアイコンに畳まれ、タップで出席へ
- **裏処理の最適化**: CLASS画面の裏起動は「授業時間帯」または「出席画面を開いている間」だけに限定し、電池・セッション負荷を抑制
