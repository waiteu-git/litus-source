# セキュリティポリシー / Security Policy

## 対象バージョン / Supported Versions

配布中の最新バージョン、および公開ミラー [litus-source](https://github.com/waiteu-git/litus-source) の `main` ブランチの最新コードを対象とします。

## 脆弱性の報告 / Reporting a Vulnerability

**公開 Issue には書かないでください。** 以下で非公開に報告をお願いします:

- GitHub の [プライベート脆弱性報告](https://github.com/waiteu-git/litus-source/security/advisories/new)（推奨・公開ミラー側で受け付けます）

報告を受けたら、確認の返信・影響評価・修正の順で対応し、修正の公開まで詳細を非公開に保ちます。善意の報告者に法的措置を取ることはありません。

## 検証時のお願い / Rules of Engagement

- **大学の本番システム（LETUS・CLASS・*.tus.ac.jp）に対する検証行為（負荷試験・脆弱性スキャン・攻撃的アクセス・出席コードの総当たり）は絶対に行わないでください。** 本プロジェクトの検証はローカル環境・自身のアカウントのデータの範囲に留めてください。
- 他人の個人情報・アカウントに触れる検証はしないでください。
- 発見した情報（セッション・個人情報等）を保存・共有しないでください。

## English

Please report vulnerabilities privately via [GitHub private vulnerability reporting](https://github.com/waiteu-git/litus-source/security/advisories/new) on the public mirror — do not open a public issue. Never test against the university's production systems (LETUS / CLASS / *.tus.ac.jp), including brute-forcing attendance codes; limit verification to your local environment and your own account's data. We will not pursue legal action against good-faith reporters.
